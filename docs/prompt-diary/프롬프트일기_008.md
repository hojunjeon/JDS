# [프롬프트 일기 #8] 사용자의 학습 맥락을 AGENTS.md에 반영하기
> JDS를 만들면서 AI와 함께 일하는 방식을 기록한다.
> 어떤 프롬프트가 들어왔고, 무엇이 바뀌었고, 그래서 어떤 결정을 남겼는지 적는다.

---

## 그 시점의 게임

JDS는 게임 자체를 만드는 프로젝트이면서, 사용자가 Codex 사용법과 개발 협업 습관을 익히는 실습장이기도 하다. 기존 `AGENTS.md`에는 작업 범위, 검증, 스킬 사용, Git 규칙은 정리되어 있었지만, 사용자의 현재 학습 수준과 답변 톤에 대한 규칙은 명시되어 있지 않았다.

---

## 그날 상황

사용자는 자신이 전문 게임 개발자가 아니라, Codex 사용법을 게임 개발로 익히려는 개발자 취업 준비생이라고 설명했다. 그래서 앞으로 Codex가 답변할 때 이 맥락을 이해하고, 너무 전문가 기준으로만 말하지 않도록 `AGENTS.md`를 조정하고 싶다고 했다.

---

## 입력된 프롬프트

> "나는 전문적인 게임 개발자가 아니라 codex 사용법을 게임 개발로 익히려는 개발자 취준생이야. 이걸 이해하고 답변은 이 수준에 맞게 할 수 있도록 AGENT.md를 수정하고 싶어."

---

## 무엇이 변화했나

`AGENTS.md`에 `User Context and Response Style` 섹션을 추가했다. 핵심은 사용자를 전문 게임 개발자가 아니라 Codex와 게임 개발을 함께 배우는 개발자 취업 준비생으로 보라는 것이다.

답변 방식도 함께 정리했다. 낯선 게임 개발 용어, 엔진 개념, 테스트 습관, 워크플로우 판단은 필요할 때 짧게 설명하고, 변경을 제안하거나 구현할 때는 왜 그렇게 하는지까지 알려주도록 했다. 또한 한국어 답변을 기본으로 하고, 작업을 취업 준비에 도움이 되는 학습 가치와 연결하도록 했다.

---

## 그래서 어떤 결정이 생겼나

앞으로 JDS에서 Codex는 단순히 결과물을 만드는 도구가 아니라, 사용자가 개발 사고방식과 AI 협업 방식을 배울 수 있도록 돕는 페어 프로그래밍 파트너에 가깝게 행동한다.

다만 설명이 길어져 작업 흐름을 방해하지 않도록, 답변은 간결하고 실행 중심으로 유지한다. 즉, "초보자에게 친절하되 장황하지 않기"가 새 기준이다.

---

## 다음 중 보고 싶은 것

이 규칙이 실제 작업에서 잘 작동하는지 봐야 한다. 앞으로 답변이 너무 어렵거나, 반대로 너무 기초 설명에 치우치면 `AGENTS.md`의 문장을 다시 조정하면 된다.
