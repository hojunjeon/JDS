# [프롬프트 일기 #10] UI 리뉴얼을 DOM overlay 중심으로 다시 정렬

> JDS를 만들면서 Codex와 어떻게 작업하는지 기록한다.
> 어떤 프롬프트가 들어왔고, 무엇이 바뀌었고, 그래서 어떤 결정이 생겼는지 적는다.

---

## 그 시점의 게임

UI Renewal Phase 1은 기능 흐름 기준으로는 완료되어 있었다. Start, Stage Select, Weapon Select를 거쳐 gameplay까지 들어갈 수 있었고, `menuFlow.ts` 테스트와 Playwright smoke check도 있었다.

하지만 실제 화면은 `codex_examples/01`, `02`, `03`의 HTML reference와 많이 달랐다. 기존 구현은 Phaser canvas에 텍스트와 사각형을 직접 그린 간단한 production pass였고, 사용자가 마음에 들어 한 reference의 3열 IDE layout, explorer, tabs, code preview, briefing panel 같은 특징은 충분히 살아나지 않았다.

---

## 그날 상황

사용자는 `codex_examples/`의 UI 디자인이 마음에 들고, 최대한 유사하게 구현하고 싶다고 했다. Codex는 Phaser UI와 DOM overlay의 차이를 설명했고, 메뉴와 결과/도감처럼 정적인 화면은 DOM overlay가 reference와 더 잘 맞고, 인게임 HUD처럼 전투 중 계속 갱신되는 화면은 Phaser가 더 안전하다고 정리했다.

> "좋아 해당 결정을 반영해서 phase 1 수정 계획 진행하고 나머지 phase 들의 계획 수정할 것이 있다면 수정해줘"

## 뭐가 변화했나

Phase 1은 더 이상 "기능 흐름 완료"만으로 완료 상태로 보지 않기로 했다. 기능 흐름은 완료됐지만, 시각 정렬은 다시 열었다.

새 실행 계획 `docs/superpowers/plans/2026-04-27-ui-renewal-phase-1-dom-menu-alignment.md`를 추가했다. 이 계획은 `menuFlow.ts`를 단일 상태 모델로 유지하고, `MenuScene`은 scene lifecycle과 gameplay handoff를 맡으며, 실제 Start / Stage / Weapon UI는 vanilla TypeScript DOM overlay와 CSS로 구현하는 방향이다.

`DESIGN.md`에는 `codex_examples/01-10`이 현재 UI renewal reference라는 점과 DOM/Phaser split 기준을 추가했다. `PLANS.md`와 `the former UI Renewal roadmap`도 Phase 1 DOM visual alignment가 현재 작업임을 반영했다.

Phase 2, Phase 3, Phase 4 계획도 수정했다. Phase 2는 Phaser HUD + DOM event/boss alert의 hybrid 방식이 되었고, Phase 3과 Phase 4는 DOM overlay 중심 계획으로 바뀌었다.

---

## 그래서 어떤 결정이 있었나

앞으로 UI Renewal은 다음 기준을 따른다.

- 정적이고 layout-heavy한 화면은 DOM overlay로 구현한다.
- 전투 중 계속 갱신되거나 gameplay timing에 가까운 UI는 Phaser로 구현한다.
- DOM overlay는 gameplay 진입 또는 resume 전에 반드시 제거한다.
- Playwright는 canvas 표시만 보지 않고 DOM selector와 screenshot을 함께 검증한다.

이 결정은 사용자가 reference HTML의 디자인을 실제 게임 UI에 더 강하게 반영하고 싶다는 방향을 명확히 한 결과다.

---

## 다음 중 떠오른 것

이 변경은 구현보다 verification 습관이 중요하다. DOM overlay를 쓰면 화면은 더 예뻐지기 쉽지만, keyboard input 중복, cleanup 누락, canvas 위에 남는 invisible layer 같은 문제가 생길 수 있다. Phase 1 구현 때는 `.jds-menu-root`가 gameplay 시작 후 사라졌는지 e2e에서 반드시 확인해야 한다.
