# [프롬프트 일기 #11] PLANS.md를 활성 phase router로 정리하기

> JDS를 만들면서 Codex와 어떻게 작업하는지 기록한다.
> 어떤 프롬프트가 들어왔고, 무엇이 바뀌었고, 그래서 어떤 결정이 남았는지 적는다.

---

## 그 시점의 게임

UI Renewal은 여러 phase 계획으로 나뉘어 있었다. 하지만 루트 `PLANS.md`, 옛 roadmap 문서, phase별 실행 계획이 함께 남아 있으면 다음 Codex 세션이 어떤 문서를 기준으로 재개해야 하는지 헷갈릴 수 있었다.

---

## 그날 상황

사용자는 `PLANS.md`를 유일한 활성 phase router로 만들고, 상세 진행은 `docs/superpowers/plans/` 안의 phase별 체크박스를 기준으로 삼으라고 요청했다. 완료된 큰 작업은 루트 라우터에 계속 남기지 않고 archive로 옮기는 규칙도 추가했다.

---

## 입력된 프롬프트

> "PLANS.md becomes the only active phase router. Detailed task progress is read from docs/superpowers/plans/*.md checkboxes."

---

## 무엇이 변화했나

`PLANS.md`는 이제 `Active work: UI Renewal`과 phase 목록만 보여준다. `[~]` 표시는 phase row에만 쓰고, 세부 단계는 phase plan 안의 첫 번째 미체크 항목에서 찾는다.

옛 roadmap 파일은 삭제 대상으로 정리했고, review deck과 prompt diary의 정확한 옛 파일명 참조도 제거했다. `docs/superpowers/plans/ARCHIVE.md`는 완료된 큰 작업을 나중에 옮길 장소로 추가했다.

---

## 그래서 어떤 결정이 남았나

앞으로 세션이 시작되면 `PLANS.md`를 먼저 읽고, `[~]` phase plan을 연 뒤, 그 plan의 첫 번째 `[ ]` step부터 이어간다. `PLANS.md`와 상세 plan이 충돌하면 상세 plan을 신뢰한다.

---

## 다음에 떠올릴 것

큰 작업이 끝났을 때는 `PLANS.md`에 완료 항목을 오래 남겨두지 않는다. 검증이 끝난 요약은 archive로 옮기고, active router에는 지금 이어서 할 일만 남기는 편이 Codex 재개 비용을 줄인다.
